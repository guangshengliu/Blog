Floyd

1. Introduction to Floyd algorithm

Floyd, also known as interpolation, is an algorithm that uses the idea of dynamic programming to find the shortest path between multiple source points in a given weighted graph, similar to Dijkstra's algorithm. The algorithm is named after Robert Freud, one of the founders, a winner of the Turing Award in 1978, and a professor of computer science at Stanford University. The algorithm finds the shortest path length between any two points u and V for a given graph G (V, E) with a time complexity of O (n^3).Since the complexity of n^3 determines the limit of vertex number n to less than 200, it is appropriate and convenient to implement the Floyd algorithm using an adjacency matrix.

2. Floyd algorithm idea

Algorithmic process:

  1. Start with any one-sided path. The distance between all two points is the weight of the edge, and if there is no edge connecting between the two points, the weight is infinite.
  2. For each pair of vertices u and v, see if there is a vertex w that makes the path from u to w to V shorter than known. If it is updated. The Floyd algorithm is based on the fact that if there is vertex k, which shortens the current distance between vertex i and vertex J when k is used as the intermediate point, vertex K is used as the intermediate point between vertex i and j, that is, when dis[i] [j] + dis[k] [j] < dis[i] [j], dis[i] = dis[i] [k] + dis[k] [j] (where dis[i] [j] represents the shortest distance from vertex i to vertex j).

3. Floyd algorithm implementation

Based on the above facts, the implementation code of the Floyd algorithm is as follows:

const int INF = 100000000;
const int MAXV = 200;   //MAXV is the maximum number of vertices.
int n,m;        //N is the number of vertices and M is the number of edges.
int dis[MAXV][MAXV];    //dis[i][j] represents the shortest distance between vertex i and j.

void Floyd(){
    for (int k = 0; k < n; k++){
        for (int i = 0; i < n; i++){
            for (int j = 0; j < n; j++){
                if(dis[i][k]!=INF&&dis[k][j]!=INF&&dis[i][k]+dis[k][j] < dis[i][j]){
                    dis[i][j] = dis[i][k] + dis[k][j];  //Find a shorter path.
                }
            }           
        }       
    }    
}

For the Floyd algorithm, it is important to note that the outermost K loop cannot be put into the inner layer (That is, a triple loop of i->j->k is generated) which will result in an error in the final result. The reason is: if dis[u] [v] accessed later is optimized, dis[i] [j] accessed earlier will not be further optimized because it has already been accessed (i, J is accessed before u, V here).
